An application that makes researching, evaluating and comparing recommendation algorithms easy

Implicit feedback collaborative filtering recommendation algorithms

Simulating and comparing recommendation algorithms

Popularity





This algorithm doesn't give personal recommendations. Every user gets the top-k most popular items of that moment recommended. The parameters are: window size (for popularity calculation) and retrain interval (to decide when the top-k is recalculated again).

Recency





Like the popularity algorithm, the same list is shown to all users, but now based on the most recent items. Recency is based on the first interaction with an item (to estimate when the item was introduced). Parameter: retrain interval.

ItemKNN





The item K-nearest-neighbours algorithm uses a heuristic to determine the "similarity" between items. Next, a personalized recommendation list gets put together for each user based on items that are the most similar to the items in the user's history. Parameters: K, window, normalize and retrain interval.

About Us

In March 2022, we started Recommended4You as a project for the course Programming Project Databases. Our team consists of five students at the University of Antwerp (Belgium).

With this application, we try to provide an easy way to simulate recommendation algorithms. The idea behind the recommendation algorithms is simple: we want to give a user recommendations based on his history of searched items (implicit feedback), using the history of other similar users (collaborative filtering).
The goal of this project is to provide a "recommendation simulator and explorer".
